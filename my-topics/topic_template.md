# 題目構想

## 題目名稱

航運資料儀表板：用 Gradio 建立互動式航運資料分析與視覺化儀表板

## 題目動機

航運是全球供應鏈與國際貿易的重要環節，港口壅塞、船舶延誤、貨櫃量變化與船舶週轉效率，都會影響物流效率與港口營運表現。

然而，這些資料通常分散在大量表格或 CSV 檔案中，若只用原始資料查看，不容易快速判斷港口營運狀況。因此，本專題希望透過 Python 進行資料清洗、統計分析與視覺化，並使用 Gradio 建立互動式 Web 儀表板，讓使用者可以更直覺地觀察航運資料重點。

本題目可以整合本學期所學的資料分析、資料視覺化、互動式介面、Docker 容器化與 GitHub 版本控制，完整呈現從資料處理到應用部署的流程。

## 為什麼對這個題目有興趣？

航運資料與海事資訊科系具有高度相關性，透過這個題目可以把課堂所學的 Python 資料分析技術應用在航運與港口營運情境中。

本專題不只是製作圖表，而是希望建立一個能讓一般使用者操作的資料儀表板。使用者可以上傳 CSV 檔案，系統自動清洗資料、產生統計結果，並透過圖表呈現港口吞吐量、平均滯港時間、船速與延誤率等資訊。

透過這個專題，可以練習資料處理、視覺化設計、Web 介面建置與 Docker 部署，也能展現資訊技術在航運資料分析中的應用價值。

## 預計使用的技術

| 技術               | 是否使用 | 用途              |
| ---------------- | ---- | --------------- |
| Python           | 是    | 專案主要開發語言        |
| Pandas           | 是    | 資料讀取、清洗、轉換與統計分析 |
| NumPy            | 是    | 數值運算與缺失值處理      |

## 預計系統功能

* 上傳航運 CSV 資料
* 使用內建模擬資料進行展示
* 自動檢查必要欄位
* 自動進行資料清洗
* 移除重複資料與不合理數值
* 將時間欄位轉換為日期時間格式
* 依港口、航線、船舶類型進行篩選
* 顯示港口吞吐量
* 顯示平均滯港時間
* 顯示平均航速
* 顯示船班延誤率
* 產生港口績效比較圖
* 產生航線績效比較圖
* 產生時間趨勢圖
* 將清洗後資料輸出到 `data/processed/`

## 預期成果

本專題預期完成一個可在瀏覽器操作的航運資料儀表板。使用者可以透過 Gradio 介面上傳 CSV 檔案，系統會自動完成資料清洗、統計分析與圖表產生。

儀表板可依照港口、航線與船舶類型篩選紀錄，並以表格與圖表呈現港口吞吐量、平均滯港時間、平均航速與延誤率等重點資訊。

最後，本專題會透過 Docker 進行容器化部署，並將完整程式碼、資料說明、專題提案與報告文件上傳至 GitHub，展現資料處理、視覺化、互動介面與容器化部署的整合能力。
