# 題目構想

## 題目名稱

航運資料儀表板：用 Gradio 建立互動式航運資料分析與視覺化儀表板

## 為什麼對這個題目有興趣？

航運是全球供應鏈的重要環節，但港口壅塞、船舶延誤與貨櫃量波動往往分散在大量表格資料中，不容易直接判讀。這個題目可以把資料清洗、統計分析、視覺化與雲端部署串成完整流程，也能展示如何把 Python 分析成果轉成一般使用者可操作的 Web 儀表板。

## 可能使用的資料來源

| 資料名稱 | 來源網址 | 資料格式 |
| --- | --- | --- |
| 模擬航運與港口物流資料 | 專題自行建立 | CSV |
| 臺灣航港公開資料 | 交通部或航港局公開資料平台 | CSV / API |
| 國際航運指數資料 | 上海航運交易所、Baltic Exchange 等公開資訊 | CSV / API |

## 預計使用的技術

- [x] Python + Pandas 資料分析
- [x] Matplotlib 視覺化
- [x] Jupyter Notebook
- [x] Gradio 互動介面
- [x] Docker 容器部署
- [ ] Keras 預訓練模型
- [ ] MySQL 資料庫

## 預期成果

完成一個可在瀏覽器操作的航運資料儀表板，能上傳 CSV、清洗資料、依港口與航線篩選紀錄，並用表格與圖表呈現港口吞吐量、滯港時間、航速與延誤率。
