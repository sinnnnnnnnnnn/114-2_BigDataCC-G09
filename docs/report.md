## 三、摘要

本專題以航運與港口物流資料為主題，建立一套從 CSV 資料清洗、統計分析、視覺化到互動式儀表板展示的完整流程。

系統使用 Pandas 處理資料品質問題，包含時間格式轉換、缺失值補齊、異常值過濾與重複紀錄移除；再以 Matplotlib 繪製港口與航線績效圖表，最後透過 Gradio 提供瀏覽器介面，讓使用者能夠切換港口、航線、船舶類型與分析指標。

本專題也使用 Docker 進行容器化部署，使應用程式可以在不同環境中執行，並透過 GitHub 管理專案程式碼與文件。

## 四、專題動機與目標

航運資料通常包含多個時間點、多個港口與多種船型的營運紀錄，例如船舶到港時間、貨櫃吞吐量、滯港時間、平均航速與延誤狀態等。

如果只依靠原始表格，使用者難以快速比較港口壅塞程度、航線吞吐量與延誤狀況。因此，本專題目標是將資料分析流程包裝成互動式工具，讓使用者不需要撰寫程式，也能透過視覺化方式掌握航運與港口營運概況。

本專題主要目標如下：

1. 建立航運資料清洗流程。
2. 使用 Pandas 與 NumPy 進行統計分析。
3. 使用 Matplotlib 產生資料視覺化圖表。
4. 使用 Gradio 建立互動式 Web 儀表板。
5. 使用 Docker 進行容器化部署。
6. 使用 GitHub 管理專案程式碼與文件。

## 五、資料集說明

本專題使用的資料位於：

```text
data/raw/sample_shipping_data.csv
```

該資料為專題自行建立的模擬航運資料，共 60 筆紀錄，涵蓋多個港口與航線，例如 Singapore、Shanghai、Tokyo、Los Angeles、Rotterdam，以及 Asia-Europe、Trans-Pacific、Intra-Asia 等航線。

### 5.1 資料欄位

| 欄位                 | 說明          |
| ------------------ | ----------- |
| `timestamp`        | 船舶到港或作業紀錄時間 |
| `port_name`        | 港口名稱        |
| `route`            | 航線區域        |
| `vessel_imo`       | 船舶 IMO 識別碼  |
| `vessel_type`      | 船舶類型        |
| `turnaround_hours` | 船舶滯港或週轉時間   |
| `cargo_teu`        | 貨櫃吞吐量       |
| `avg_speed_knots`  | 平均航速        |
| `delay_status`     | 準點或延誤狀態     |

### 5.2 資料用途

本資料集主要用於模擬港口與航運營運情境，讓系統可以進行資料清洗、統計分析與視覺化展示。透過此資料，使用者可以觀察不同港口、航線與船舶類型的營運表現。

## 八、資料分析過程

### 8.1 資料清洗

系統首先檢查必要欄位是否存在，確保上傳的 CSV 檔案能夠正確進行分析。接著，系統會將 `timestamp` 欄位轉換為 datetime 格式，並將 `turnaround_hours`、`cargo_teu`、`avg_speed_knots` 等欄位轉換成數值格式。

若資料中出現缺失值，系統會先使用同港口的中位數補齊；若仍無法補齊，則使用整體中位數處理。系統也會移除時間無法解析、滯港時間為負、航速超過合理範圍、貨櫃量異常或重複的紀錄。

清洗後的資料會輸出至：

```text
data/processed/cleaned_shipping_data.csv
```

### 8.2 統計分析

分析模組會產出整體 KPI 摘要，包含：

* 資料筆數
* 總 TEU
* 平均滯港時間
* 平均航速
* 延誤率

此外，系統也會依照港口與航線進行分組分析，計算各港口與航線的吞吐量、平均滯港時間與延誤比例，協助使用者比較不同港口與航線的營運表現。

### 8.3 資料視覺化

儀表板提供以下三種主要圖表：

| 圖表名稱                    | 說明          |
| ----------------------- | ----------- |
| Trend Line Chart        | 觀察指定指標的每日變化 |
| Port Average Bar Chart  | 比較不同港口的平均表現 |
| Route Average Bar Chart | 比較不同航線的平均表現 |

http://localhost:8080
