## 三、專題動機

航運與港口營運資料通常包含船舶進出港、貨櫃吞吐量、平均航速、滯港時間與延誤狀態等資訊。這些資料若只以 CSV 或一般表格保存，使用者不容易快速判斷哪個港口較壅塞、哪條航線延誤較嚴重，或貨櫃量是否集中於特定航線。

因此，本專題希望使用 Python 建立一套簡單但完整的航運資料分析流程，從資料讀取、資料清洗、統計分析到圖表視覺化，最後再透過 Gradio 建立互動式儀表板。透過此系統，非技術使用者也可以上傳航運 CSV 資料、切換篩選條件，並即時查看分析結果與視覺化圖表。

本專題也會使用 Docker 將應用程式容器化，使系統能夠在不同環境中快速部署與執行，並使用 GitHub 進行版本控制與成果管理。

## 八、系統功能規劃

本專題預計完成以下功能：

1. 使用內建模擬航運資料進行分析。
2. 支援使用者上傳 CSV 檔案。
3. 自動檢查 CSV 是否包含必要欄位。
4. 自動清洗資料，例如處理缺失值、重複值與不合理數值。
5. 將 `timestamp` 欄位轉換為時間格式。
6. 依港口、航線與船舶類型進行篩選。
7. 顯示 KPI 摘要，例如總貨櫃量、平均滯港時間、平均航速與延誤率。
8. 顯示港口績效表。
9. 顯示航線績效表。
10. 使用 Matplotlib 產生趨勢圖與比較圖。
11. 使用 Gradio 建立互動式 Web 介面。
12. 使用 Docker 進行容器化部署。

## 十、預期成果

本專題完成後，使用者可以在瀏覽器中開啟航運資料儀表板，使用系統內建資料或自行上傳 CSV 檔案，並透過港口、航線、船舶類型與最低貨櫃量等條件篩選資料。

系統會即時顯示 KPI 摘要、港口績效、航線績效與視覺化圖表，協助使用者快速了解港口營運狀況、航線表現與船班延誤情形。

最後，本專題將使用 Docker 進行容器化部署，並將完整程式碼、資料集說明、專題提案、報告與測試檔案上傳至 GitHub，展現本學期資料分析、視覺化、互動式應用與容器化部署的整合能力。
